BurgerLab Documentation

Everything you need to ship.

Thanks for purchasing BurgerLab — a premium burger & restaurant HTML template. This guide covers setup, customization and every feature, with copy-paste snippets throughout.

Introduction

BurgerLab is a static HTML5/CSS3/JavaScript template — no build server or database required. It ships with 6 homepages, 10 header and 10 footer layouts, 30+ inner pages, a complete SCSS design-token system, GSAP-powered motion, custom date & time pickers, and serverless forms via Web3Forms.

You can use it exactly as-is (just edit the HTML and images), or recompile the SCSS for deeper theming. Both workflows are documented below.

New here? Jump to Quick start to preview the template locally in under a minute.

What's included

  • 6 home variants — Cinematic, Slider, App, Split, Statement and Bento (index.html … index-6.html).
  • 10 header layouts and 10 footer layouts, each a single modifier class (showcased in headers.html / footers.html).
  • 20 inner pages — menu, menu by category, inside-the-burger spotlight, about, the team, services + service detail, gallery, journal + article, shopping cart, checkout, contact, reservation, single product, FAQ, 404 and coming soon, plus privacy policy and terms.
  • Landing/demo page (landing/) and this documentation.
  • Source SCSS with a full design-token system, plus the compiled style.css.
  • Vendor libraries — GSAP + ScrollTrigger, Swiper, Fancybox (self-hosted in assets/js/vendors/).
  • Custom date & time pickers and Web3Forms integration — no jQuery, no backend.

Quick start

The template is plain HTML — open any .html file in a browser to preview it. Because some assets load over fetch/relative paths, it's best to run a tiny local server rather than opening files directly:

# From the template root, pick any one:
python3 -m http.server 8000        # Python 3
npx serve .                        # Node (npx)
php -S localhost:8000              # PHP

Then visit http://localhost:8000/ for the home page, or /landing/ for the demo overview.

You only need Node/Sass if you want to recompile the SCSS. To edit content and colors inline, plain HTML/CSS edits are enough.

SCSS workflow

Styles are authored in assets/scss/ and compiled to assets/css/style.css. Edit the partials, then compile with the Dart Sass CLI (no install needed via npx):

# One-off build
npx sass assets/scss/style.scss assets/css/style.css --style=expanded --no-source-map

# Watch & rebuild on save
npx sass --watch assets/scss/style.scss assets/css/style.css --no-source-map

Never edit style.css directly — it's generated and your changes will be overwritten on the next compile.

Colors & theme

The palette lives in assets/scss/_tokens.scss as Sass variables, and is also exposed as CSS custom properties on :root so you can override colors at runtime. Edit the variable, recompile, and the whole template updates.

TokenValueRole
--bl-bg#10100FPage background
--bl-bg-2#181816Alternate section bg
--bl-surface#20201DCards / surfaces
--bl-surface-2#2A2924Raised surfaces
--bl-text#F8F4EAPrimary text
--bl-muted#A9A193Muted text
--bl-accent#F5B31ABrand amber (CTAs)
--bl-accent-2#FF6A2AFlame orange
--bl-green#4CD36FSuccess / positive
--bl-line#35332DBorders / dividers

Option A — recompile (recommended)

// assets/scss/_tokens.scss
$bl-accent: #F5B31A;   // change to your brand color
// then: npx sass assets/scss/style.scss assets/css/style.css --no-source-map

Option B — runtime override (no build)

Add a small stylesheet or inline block after style.css and override the custom properties:

<style>
  :root { --bl-accent: #E23744; --bl-accent-2: #FF8A3D; }
</style>

Typography

Three Google Fonts are used: Space Grotesk (headings), Inter (body) and JetBrains Mono (labels/eyebrows). They're loaded via the <link> in each page's <head> and mapped to tokens in _tokens.scss:

$bl-font:      "Inter", system-ui, sans-serif;       // body
$bl-font-head: "Space Grotesk", system-ui, sans-serif; // headings
$bl-font-mono: "JetBrains Mono", ui-monospace, monospace;

To change a font: update the Google Fonts <link> in the <head> of each page (or your includes), then change the matching variable and recompile.

Logo & favicon

The logo is text-based for crispness at any size — edit it directly in the header markup:

<a href="index.html" class="bl-logo">Burger<span>Lab</span></a>

To use an image logo, replace the inner text with an <img>. The favicon lives at assets/images/favicon.png and is referenced in each page's <head> — swap the file (keep the name) or update the path.

Components

Reusable UI lives in _buttons.scss and _components.scss. The most-used pieces:

<!-- Buttons -->
<a class="bl-btn bl-btn--primary">Order now</a>
<a class="bl-btn bl-btn--ghost bl-btn--sm">View menu</a>
<a class="bl-btn bl-btn--flame bl-btn--lg">Reserve</a>

<!-- Section heading -->
<div class="bl-section-head bl-section-head--center">
  <span class="bl-eyebrow">The menu</span>
  <h2 class="bl-section-title">Signature <em>burgers</em></h2>
</div>

Button modifiers: --primary, --flame, --ghost, --light, plus sizes --sm / --lg and --block. Layout helpers include .bl-container(--wide/--narrow), .bl-section(--bg2/--surface) and .bl-grid--2/3/4.

The FAQ accordion (see faq.html) needs no JS at all — it's native <details>/<summary> styled with .bl-faq / .bl-faq__item:

<details class="bl-faq__item" open>
  <summary>Do you deliver?</summary>
  <div class="bl-faq__a"><p>Yes — 25–35 minutes across the city.</p></div>
</details>

The coming-soon.html page includes a live countdown — add data-countdown="2026-08-15T11:00:00" to a container with four [data-cd="days|hours|minutes|seconds"] children, and initCountdown() in main.js updates them every second.

Date & time pickers

BurgerLab ships brand-themed calendar and analog-clock pickers (vanilla JS, no plugins) — see them on reservation.html. Wrap a text input in a data-datepicker or data-timepicker element:

<div class="bl-picker" data-datepicker>
  <input class="bl-input" type="text" name="date" placeholder="Select a date" required>
  <span class="bl-picker__icon">…calendar svg…</span>
</div>

The script (in main.js) shows a human value in the field (e.g. Jun 14, 2026 / 7:30 PM) and stores a machine value in input.dataset.value (ISO date / 24-hour time) for reliable parsing. Past dates are disabled automatically.

Forms & Web3Forms

The contact and reservation forms submit through Web3Forms — serverless, no backend. To receive submissions, create a free access key and replace the placeholder in each form:

<form action="https://api.web3forms.com/submit" method="POST" data-form>
  <input type="hidden" name="access_key" value="YOUR-ACCESS-KEY-HERE">
  …
</form>

Replace the access key in both contact.html and reservation.html. The form handler in main.js manages validation and success/error states for you.

Sliders & lightbox

Swiper powers the home-v2 hero carousel and the reviews slider; Fancybox powers the gallery lightbox. Both are self-hosted in assets/js/vendors/ and initialized in main.js (initHeroSlider, initReviewsSlider, initFancybox). Add a lightbox to any image with:

<a data-fancybox="gallery" href="assets/images/your-photo.jpg">
  <img src="assets/images/your-photo.jpg" alt="…">
</a>

Animations

Scroll animations use GSAP + ScrollTrigger. Add reveal behavior with utility classes — no JS required:

<div class="bl-reveal-up">Slides up on scroll</div>
<div class="bl-reveal-fade">Fades in on scroll</div>
<div data-stagger>
  <div>Child 1</div><div>Child 2</div> <!-- staggered -->
</div>

All motion respects prefers-reduced-motion — when a visitor opts out, content renders statically. If GSAP is missing, the page still displays correctly (graceful fallback).

RTL support

A right-to-left stylesheet (_rtl.scss) is compiled into the main CSS. Enable it by setting the document direction:

<html lang="ar" dir="rtl">

Layout, spacing and nav mirror automatically for Arabic, Hebrew and Persian content.

Deployment

Because the template is fully static, deployment is just uploading the files. It works on any static host — Netlify, Vercel, Cloudflare Pages, GitHub Pages, or classic shared hosting via FTP.

  • Upload the entire template folder (keep the assets/ structure intact — paths are relative).
  • Set index.html as the default document.
  • Optionally remove documentation/ and landing/ from your production upload.
  • Recompile style.css first if you changed any SCSS.

Browser support

Tested on the latest versions of Chrome, Edge, Firefox and Safari (desktop & mobile), and responsive from 320 px to 1920 px. The template uses modern CSS (grid, custom properties, clamp()) supported by all current evergreen browsers.
